# 토스 앱 UI 해체분석 (Discovery)

- 작성일: 2026-05-15
- 작성자: Claude (analysis session)
- 분류: 외부 UI 디스커버리 / 디자인 리서치
- 권위: **본 레포의 구현 권위가 아님**. 외부 앱(토스/토스증권/토스뱅크) UI를 해체분석하여 추후 디자인 의사결정에 참조하기 위한 노트.

## 목적

토스 앱 9장 스크린샷을 입력으로, UI를 그대로 복제하지 않고 **그 뒤에 깔린 기준·의도·정책·철학·디테일**을 추출한다. 그 다음, 추출한 규칙만으로 원본 스크린의 골조를 재구성 가능한지 시뮬레이션하여 추출 결과의 정합성을 검증한다.

## 입력 스크린

| # | 화면 | 핵심 패턴 |
|---|---|---|
| 1 | 토스증권 홈 | 타이틀 + 인라인 지수 · 캠페인 카드 · 개인화 추천 리스트 |
| 2 | 거래대금 차트 | 스크롤 시 상단 지수 슬롯 교체(VIX) · 광고+랭킹 리스트 · 모드 토글 pill |
| 3 | 관심(빈 상태) | AI 시그널 배너 + 친절한 빈 상태(이름 호명 + 액션 버튼) |
| 4 | 발견 | 카테고리 칩 · AI 이슈 outlined 카드 · 5-탭 세그먼트 |
| 5 | 피드 | 4-탭 세그먼트 · 작성자+뱃지+토픽+팔로우가 한 줄 · 본문 첫 줄 굵게 |
| 6 | 전체 메뉴 | 이름 호명 + 검색 · 바로가기 그리드 · 카테고리 리스트(아이콘+메타) |
| 7 | 설정 | 프로필 헤더 · 그룹 카드(라운드 컨테이너) 구조 |
| 8 | 통장 메인 | 잔액 hero · 이자 인라인 CTA · 최근 거래 미리보기 · 듀얼 CTA |
| 9 | 거래내역 | 잔액 중앙 hero · 풀-와이드 이자 CTA · 날짜 그룹 + 광고 슬롯 · 스티키 듀얼 CTA |
| 10 | 계좌 상세 | 계좌 헤더+복사 · 주요값 인라인 · "내 계좌 관리/이체 관리" 그룹 카드 |

(스크린샷 자체는 본 레포에 저장하지 않음. 분석 시 채팅 첨부로 전달됨.)

## 1. 디자인 시스템 토큰 추출

### 1.1 컬러

| 토큰 | 추정값/역할 | 사용처 |
|---|---|---|
| `bg/base` | 거의 순흑(#0E0F12쯤) | 모든 화면 바탕 |
| `bg/surface` | 한 단계 밝은 회흑(#1B1D22쯤) | 카드, 그룹 컨테이너, 입력창 |
| `bg/elev` | 그보다 살짝 밝음 | pill 토글, 칩, 작은 버튼 |
| `text/primary` | 순백 | 잔액·종목명·타이틀 |
| `text/secondary` | 중간 회색 | 부가설명, 메타 |
| `text/tertiary` | 어두운 회색 | 면책 문구, placeholder |
| `brand/blue` | 토스 시그니처 블루(#3182F6 계열) | 1차 CTA, 링크, 활성 탭, 랭킹 숫자, "하락"(한국식) |
| `semantic/up` | 레드(#F04452 계열) | "상승" — 한국 증권 관행 |
| `semantic/down` | 블루(브랜드 블루 같은 톤) | "하락" |
| `ai/accent` | 라벤더/퍼플 | ✨ AI 마커, AI 카드 outline |
| `product/mint` | 민트그린 | 토스뱅크/용돈/특정 서비스 아이콘 한정 |

**관찰**
- 브랜드 블루 1개로 "1차 액션 / 링크 / 하락 / 랭킹 숫자"까지 다 떠받친다. 다른 의미의 파랑이 별도로 없다 → 신호 과부하 위험을 감수하고 '브랜드 색 일관'을 택함.
- AI는 보라계열로 명확히 분리. 광고는 색이 아니라 라벨("AD", "서비스·광고")로 분리. 즉 **기능적 색**과 **마케팅적 색**을 분리.

### 1.2 타이포

- 한글: Pretendard로 추정되는 기하학적 산세리프. 가변(Variable) 웨이트 활용.
- 위계: **Hero(36~44pt Bold) · Title(24~28 Bold) · Body(16 Regular/Medium) · Caption(13~14)**.
- 숫자 비중이 매우 큼: 잔액/지수/가격이 거의 항상 그 화면의 최대 글자.
- 본문 강조는 **첫 문장만 Bold** 같은 미세 규칙 사용(피드 게시글).

### 1.3 형태/간격

- 코너 라운드: 작은 칩 ~12, 카드 ~16~20, 그룹 컨테이너 ~24, CTA 버튼 풀라운드(pill).
- 카드 패딩: 좌우 ~20, 상하 ~20~24 — 답답함보다 여백을 일관되게 우선.
- 리스트 row: 비교적 큰 64~72pt 높이. 손가락 친화.
- 세이프 에어리어 위에 떠 있는 **원형 백 버튼**과 **고정 탭바**. 백 버튼은 탭바와 같은 baseline.

### 1.4 컴포넌트 카탈로그

| 컴포넌트 | 규칙 |
|---|---|
| Top section header | 좌: 타이틀 + 인라인 라이브 컨텍스트(지수/델타) / 우: 액션 아이콘 트리오(AI ✨ · 검색 🔍 · 메뉴 ☰) |
| Primary CTA | 풀라운드, brand/blue 채움, 흰 글자 |
| Secondary CTA | 풀라운드, bg/surface, 흰 글자 |
| Pill toggle | 어두운 회색 배경 + 흰 글자(예: "간편 홈 보기") |
| Chip(필터) | 작은 라운드 + 활성 시 회색 채움(전체 ⌄) |
| Segmented tab | 텍스트 + 활성 항목 밑줄(거래대금) 또는 chip형(피드) — 컨텍스트별 다름 |
| List row | 좌 아이콘 / 중 타이틀+서브 / 우 값(또는 ❤️) / chevron |
| Ranking row | 좌측에 큰 브랜드 블루 숫자 → 강한 순위 시각화 |
| Group card | 같은 도메인 항목을 하나의 surface 카드로 묶음(설정/계좌 관리) |
| AI card | 보라 outlined 박스 + ✨ 아이콘 + 굵은 한 줄 헤드라인 |
| Promo card | 일러스트(작은 3D 풍) + 타이틀 + 서브 + CTA, 우상단 닫기 X |
| Bottom tab bar | 4~5탭, 아이콘+레이블, 활성=흰색·비활성=회색, 좌측에 백 버튼 |
| Inline cross-sell | 거래내역 row 사이에 끼워넣는 화살표+텍스트 prompt |

### 1.5 아이코노그래피

- 카테고리 아이콘은 **이모지처럼 보이는 컬러풀한 3D/스티커 풍** — 그러나 일관된 라이팅·각도. 이모지 시스템과 자체 아이콘 시스템의 경계가 흐릿하게 의도됨(친근감).
- 시스템 액션(검색/메뉴/체브론/하트)은 **모노톤 라인 아이콘**으로 통일 — 정보 vs 액션의 시각 분리.

## 2. 정책·의도·철학(추정)

### P1. "숫자가 주인공이다"
잔액·지수·가격을 화면 최대 글자로 박는다. 차트나 그래픽으로 미화하지 않고 **숫자 자체를 hero**로 둔다. 금융앱이지만 "데이터 콘솔" 같은 진지함을 유지.

### P2. "라이브 컨텍스트는 디폴트"
모든 섹션 타이틀 옆에 그 섹션과 관련된 **실시간 지수**가 인라인으로 붙는다. 증권=나스닥, 거래대금=VIX, 관심=코스피, 발견=코스닥, 피드=나스닥. 사용자는 어느 탭에 있든 "지금 시장이 어떤지" 알 수 있음. **탭과 지표의 매핑이 의도된 규칙**.

### P3. "개인화는 마케팅 카피가 아니라 골조"
"{이름}님"이 타이틀, 빈 상태, 섹션 헤더 어디든 들어간다. 빈 상태조차 이름 호명+CTA로 변환. 개인화를 **장식이 아닌 UI 컴포넌트의 슬롯**으로 취급.

### P4. "Dark-first, not dark-mode"
라이트의 다크 포팅이 아닌 다크에서 시작된 시각언어. 흰 글자에 의존하는 비중이 높고, 그라데이션·블러·일러스트가 다크 배경을 전제로 디자인됨. 라이트로 가면 오히려 어색해질 디자인 결정들이 많음.

### P5. "하나의 파랑으로 다 한다"
1차 CTA, 링크, 활성 탭, 랭킹 숫자, 하락 컬러를 모두 같은 브랜드 블루로 처리. **컬러를 의미별로 쪼개지 않고**, 컨텍스트(아이콘 모양·위치·문구)로 의미를 분리. 학습비용 ↓, 시스템 일관성 ↑, 다만 색맹/저시력 사용자에겐 까다로움.

### P6. "AI를 별도 채널로 분리"
보라 + ✨ + outlined 카드. **콘텐츠 vs AI 생성물의 시각적 출처 표시**. 신뢰 issue를 디자인으로 격리. AI가 만든 줄/말한 줄이 시각적으로 즉시 식별됨.

### P7. "광고는 숨기되 라벨링은 정직하게"
시각적 톤은 일반 카드와 동일하지만 "AD"/"서비스·광고"를 명확히 표기. 가독성을 깨지 않으면서 규제·신뢰를 충족하는 타협.

### P8. "이동을 줄이는 인라인 액션"
- 잔액 카드 안에 "이자 받기"가 박혀 있음.
- 거래내역 사이에 "안 쓰는 잔액 찾기" cross-sell.
- 차트 리스트 row 우측에 ❤️ 즉시 추가.

"다음 화면으로 가서 하는" 마찰을 최대한 깎아냄.

### P9. "빈 상태는 기능이다"
이모지 + 호명 + 1줄 본문 + 1개 CTA. 빈 상태가 **온보딩의 한 슬롯**으로 동작.

### P10. "묶기(grouping)는 시각 계층의 1차 무기"
서브헤딩 + 라운드 컨테이너 + 안쪽 row들. 설정/계좌상세에서 명확. 인지 청크 단위를 정렬.

### P11. "스크롤은 상단을 변환한다"
거래대금 화면에서 스크롤이 내려가니 상단 지수가 나스닥 → VIX로 바뀜. **타이틀 영역이 컨텍스트 변환 슬롯**으로 동작. 상단 영역을 정적 헤더가 아니라 **상태 디스플레이**로 본다.

### P12. "한국 금융 관행 존중"
- 빨강=상승, 파랑=하락 (서구 반대)
- `원` 후치, 천단위 콤마, 소수 없음
- "~님" 호칭

글로벌 시스템을 베껴 오지 않고 **로컬 컨벤션에 맞춰 토큰을 잡음**.

## 3. 재구성 시뮬레이션 — 추출한 규칙만으로 다시 만들 수 있나

각 화면을 **추출 규칙으로만** 풀어쓴 시뮬레이션 결과.

### 3-1. 증권 홈

```
[Header]
  좌: Title("토스증권") + 인라인 라이브 컨텍스트(나스닥 지수, +0.8%, 상승=red)
  우: AI ✨ · 검색 · 메뉴
[Promo Card · radius 20]
  중앙 일러스트(돈다발 3D)
  Title(2줄, Bold)
  Sub(회색)
  Primary CTA(블루 pill)
  우상단 X
[Secondary Card]
  좌: 텍스트 / 우: chevron — 채움 없는 surface 카드
[Section Header] "{이름}님을 위한 맞춤형 서비스" (P3)
[List rows × N]
  이모지풍 아이콘 / 라벨 / chevron
[Bottom Tab Bar] 증권/관심/발견/피드 + 좌측 백 버튼
```

미확정: 위쪽이 어떤 그라데이션을 쓰는지, 스크롤 시 변하는지는 정적 스샷으로 미확정.

### 3-2. 거래대금 차트

```
[Header (스크롤 변환됨)] VIX 17.26 -3.4% ← P11 적용
[AD Card] 라벨 "· AD" + 일반 카드 톤 ← P7
[Section] "실시간 거래대금 차트"
[Ranking rows × N]
  좌: 큰 블루 숫자(순위) ← P5
  중: 원형 로고 + 종목명 + 가격 + 델타(파랑/빨강) ← P12
  우: ❤️ (toggle) ← P8
[Link row] "다른 차트 보기" + chevron
[Pill toggle] "간편 홈 보기" — 모드 스위치
[Footer] 면책 문구(tertiary)
```

거의 100% 재구성 가능.

### 3-3. 관심(빈 상태)

```
[Header] Title("관심") + 코스피 지수 ← P2
[AI banner] ✨ "토스증권 AI 시그널" + 헤드라인 1줄 ← P6
[Inline tab chips] 최근 본 / 그룹추가
[Empty State]
  중앙 이모지(🔍)
  Title("{이름}님이 궁금했던 종목을 찾아보세요") ← P3, P9
  Sub("최근 본 종목이 없어요")
  Primary CTA("찾아보기")
```

완전 재구성. 빈 상태 규칙(P9)이 매우 강력하다는 검증.

### 3-4. 발견

```
[Header] Title("발견") + 코스닥
[Category chips × 4+ horizontal] 이모지 아이콘 카드
[Event widget] 좌: 텍스트 / 우: 코스피 — 인라인 라이브
[AI Card outlined] ✨ "AI 실시간 이슈" + 캐러셀 번호/제목 ← P6
[Section] "실시간 차트"
[Segmented tabs] 거래대금/거래량/급상승/급하락/인기 — 활성=밑줄
[Ranking rows × N] (3-2와 동일 컴포넌트)
```

컴포넌트가 3-2와 거의 재사용됨 → 시스템 일관성 검증 OK.

### 3-5. 피드

```
[Header] Title("피드") + 나스닥
[Segmented chip tabs] 전체(active chip) / 팔로잉 / 뉴스 / 콘텐츠 / 우측 프로필 아이콘
[Post]
  아바타 / 작성자명 / 뱃지(녹색 small pill, "5억대 자산가") / "> 토픽" / "팔로우"(블루 텍스트)
  메타("15시간 전(수정됨)")
  본문: 첫 줄 Bold ← 타이포 미세 규칙
  이미지(라운드 16) + 우하단 ⊕ floating
```

"뱃지+토픽+팔로우 한 줄"은 매우 정보밀도 높은 패턴. 추출 규칙으로 도달 가능.

### 3-6. 전체 메뉴

```
[Header] 이름 / 우측 상단 "고객센터 | 설정"
[Search bar] full-width, surface 카드형
[Section] "바로가기"
[Grid × 4] 큰 아이콘(folder형 포함) + 라벨
[Section] "최근 서비스" → 단일 아이콘
[Section] "금융 서비스"
[List rows] 큰 아이콘 + 라벨 + 우측 메타(점·구분) + chevron
[Bottom Tab Bar] 홈/혜택/쇼핑/증권/전체 ← 슈퍼앱 5탭, 현재 "전체" 활성
```

"금융서비스 row의 우측 메타"가 새로운 규칙(서브 카테고리를 점으로 나열). 정책 추가: **row 우측 메타는 점·구분으로 sub-options 노출**.

### 3-7. 설정

```
[Profile card] 아바타 + 이름 + sub("내 정보 · 주소 관리") + chevron
[Group Card · 일반]
  "언어 / 알림 / 화면 테마·진동 / 연락처 관리" — chevron rows
  "언어"만 우측에 현재값(한국어, 블루) ← P5 (블루=링크/현재 선택)
[Group Card · "인증 및 보안"]
[Group Card · "자산 및 증명서"]
```

**그룹 헤딩은 카드 바깥, 항목은 카드 안** 규칙이 뚜렷.

### 3-8. 통장 메인

```
[Top action] 좌 백 / 우 "카드, 관리"
[Hero block]
  계좌명+번호(작게, 회색 + underline 텍스트링크 느낌)
  잔액(초대형 Bold) ← P1
  하위 카드: "864원 이자 받기" Primary CTA inline ← P8
  최근 거래 3줄(날짜 약식 / 라벨 / 우측 금액 — 빨강 강조 없이 흰색 음수)
  하단 듀얼 CTA "채우기 / 보내기"
[Card] 자동이체 8개 chevron
[Card] "오늘 받을 수 있는 혜택 최소 877원" + "보기" ← P8
[Card] 용돈 0원 + 칩(+ 비상금 / + 여행비 / + 직접입력)
[Card] 미국 달러 1.00 USD + 환율 델타
```

**듀얼 CTA는 항상 페어**(채우기/보내기) 규칙 검증.

### 3-9. 거래내역

```
[Header] 백 / 카드 / 관리
[Hero] 계좌번호 underline / 잔액 중앙 정렬(초대형) / 풀-와이드 "+864원 이자 받기" pill(블루 outline 느낌)
[Filter row] "전체 ⌄" chip / 우측 검색 아이콘
[Group Header] "토스인컴 서비스·광고" ← P7
[Ad row] 큰 아이콘 + 2줄 텍스트 + chevron
[Date group] "5월 4일"
  Row: 아이콘 / 이름+시각 / 우측 금액+잔액
  Inline cross-sell card: ➡️ "투자 계좌 여러 개라면 안 쓰는 잔액 찾기" ← P8
[Sticky bottom CTA] 듀얼: 채우기(secondary) / 보내기(primary 블루)
```

**inline cross-sell은 거래 row 사이 슬롯에** 규칙 검증.

### 3-10. 계좌 상세

```
[Header] 백 / 우측 "내 계정"
[Title block] 큰 계좌별명 / sub "계좌번호·{번호}" / 우측 "복사" pill
[Info rows] 아이콘 + 라벨 + 우측 값 + chevron
[Section heading + group card] "내 계좌 관리" 외부 헤딩
[Section heading + group card] "이체 관리" — 자동이체 우측 "6건"이 블루(P5: 현재값/링크)
```

## 4. 정합성 검증 — 빈틈 / 가설로 남는 것

추출한 규칙으로 **10/10 화면 모두 골격 재구성 가능**. 그러나 정적 스샷이라 다음은 가설로 남음.

| 영역 | 미확인 | 검증 방법 |
|---|---|---|
| 모션 | 헤더 컨텍스트 전환(나스닥→VIX)이 페이드인지 슬라이드인지 | 동영상 또는 직접 사용 |
| 햅틱 | CTA·토글 햅틱 패턴 | 실기기 |
| 스크롤 시 hero shrink | 잔액 헤더가 sticky 축소되는지 | 실기기 |
| 라이트 모드 존재 여부 | 전부 다크 — 라이트 토큰 매핑이 있는지 | 설정 진입 |
| 접근성 | brand/blue 단일 사용의 색맹 보완(아이콘 형태?) | 코드 또는 a11y 감사 |
| 폰트 | Pretendard로 추정 — 실제 라이선스/대체 폰트 스택 | 실제 빌드 inspect |
| 그리드 | 컬럼 시스템(4/6/8 컬럼?)·기본 단위(4pt/8pt) | 정밀 픽셀 측정 |
| 일러스트 톤 | 3D 스티커풍의 라이팅/머티리얼 규칙 | 일러스트 가이드 |

**반례**: 거래내역 hero의 잔액이 *중앙 정렬*인데 통장 메인 hero는 *좌측 정렬*. → "**진입 깊이가 깊을수록(상세 페이지) 중앙 정렬로 hero를 더 격상시킨다**"는 추가 정책으로 해석 가능. 가설.

## 5. 종합 평가

**강점**

- 컴포넌트 재사용률이 매우 높음 (랭킹 row, 그룹 카드, 듀얼 CTA, 빈 상태 템플릿). 디자인 시스템이 잘 정착됨.
- "라이브 컨텍스트 + 개인화"를 화면 골조로 박은 게 결정적 차별점. 마케팅이 아닌 IA 차원의 결정.
- 단일 블루의 과부하를 **컨텍스트로 의미를 분리**해 우회 — 모험적이지만 일관성으로 정당화.

**약점/리스크**

- 단일 브랜드 컬러 의존 → 접근성 보조장치(아이콘/형태/굵기)에 추가 부하.
- "AD를 일반 카드 톤으로 통합" 정책은 규제 환경 변화에 취약.
- 정보 밀도가 높은 row(피드의 작성자+뱃지+토픽+팔로우)는 학습 곡선 존재.

**모방 시 주의**

- 단순히 색·폰트·라운드를 베끼면 **P1(숫자가 주인공) · P2(라이브 컨텍스트) · P3(개인화가 골조)** 이 빠져 "토스 같지 않은 토스 클론"이 됨. 진짜 차별점은 **데이터의 시각적 우선순위 결정 규칙**이지 시각 스타일이 아님.
- 토스를 베끼지 않고 "토스가 잘하는 것"만 가져가려면: **(1) hero 데이터 정의 → (2) 그 옆에 항상 붙는 라이브 컨텍스트 정의 → (3) 빈 상태를 컴포넌트화 → (4) 단일 액션 컬러로 신호 통일**.

## 6. 한 줄 결론

추출한 12개 정책(P1~P12)이 10개 화면 전체의 골조를 설명하고 재구성 가능함을 확인. 정적 스샷의 한계로 모션·햅틱·a11y 영역은 가설로 남김.

### 7.1 토스 공식: TDS(Toss Design System)

- **존재 이유**: TDS는 "토스 커뮤니티 전반에서 사용하는 공통의 디자인 언어이자 개발자의 도구"로 정의된다. 디자이너·개발자·기획자가 **같은 기준으로 협업**할 수 있도록 만든 인프라이며, 최소 품질을 보장하고 디자이너가 *"제품 자체에만 집중"* 하도록 돕는 것이 목적이다.
- **컬러 시스템(2024 전면 개편, 7년만)**: 단순 HEX가 아니라 **OKLCH/HSLuv 같은 인지적으로 균일한 색공간**을 기반으로 명도 시스템을 먼저 구축. 라이트모드와 다크모드의 명도 기준이 본질적으로 다르다는 점을 인정하고, **다크모드에 더 강한 명도 대비**를 부여. 라이트/다크는 1:1 자동 매핑.
- **시맨틱 토큰**: 색상의 값(Primitive)이 아니라 **사용 의도(Semantic)** 에 집중한 토큰 체계(`fill`/`text`/`border` × `brand`/`neutral`/`primary`/`secondary`). 동일한 의도면 같은 색이 나오게 강제하여 일관성 보장.
- **자동화**: Figma의 Token Studio → GitHub → 모든 플랫폼 코드 자동 생성. 디자인-구현 사이의 동기화 비용 제로화.
- **컴포넌트 결정 원칙**: 컴포넌트는 *"왜 존재하는가"* 가 합의되어야 추가된다. 즉 TDS는 **실험과 논증을 통과한 컴포넌트만 살아남는다**.

### 7.2 토스 공식: UX Writing 8원칙 / 5 코어밸류

토스는 **카피도 디자인 시스템의 일부**로 본다. 코어밸류로 알려진 항목:

| 원칙 | 한 줄 |
|---|---|
| **Clear** | 사용자가 한 번에 이해 가능한 단어/문장만 사용. 메이커 관점 금지. |
| **Concise** | 사용자는 안 읽는다는 전제. 필요한 만큼만, 스캔 가능하게. |
| **Casual** | 친근한 보이스톤을 **토스 정체성의 핵심 차별점**으로 본다. |
| **User-side Info** | 정보를 유저 입장에서 재구성해서 제공. |
| **Weed Cutting** | 의미 없는 문장은 잡초처럼 제거. |

### 7.4 외부 분석: "한 화면 한 기능" 원칙

여러 외부 분석가가 토스 UX를 한 마디로 요약할 때 가장 자주 인용하는 표현. 한 화면에는 사용자의 **단일 결정**만 요구하고, 보조 정보·교차판매·옵션은 다음 화면이나 인라인 슬롯으로 미룬다.

- "사용자들이 화면에 헤매는 시간을 최소한으로 줄여서 편하게 사용할 수 있도록, **한 화면에 한 기능 원칙**을 지켜서 UX를 설계"라는 식의 분석이 반복적으로 등장.

| P9 빈 상태는 기능 | ◯ | Casual + Clear 라이팅 원칙의 자연스러운 귀결. |
| P10 묶기(grouping) | ◎ | TDS 컴포넌트 가이드 작성 원칙: "구조 → 요소" 순서. 직접 일치. |
| P11 스크롤이 상단을 변환 | ─ | 외부 자료에서 매핑 못 찾음. **본 분석 고유 관찰**. |
| P12 한국 금융 관행 존중 | ◯ | 산업 표준이지 토스 고유는 아님. |

◎ 강한 일치 / ◯ 일치 / △ 부분 일치·추정 / ─ 미확인

### 7.8 외부 자료가 알려준 **새 정책**(분석에서 빠뜨린 것)

- **P0 "한 화면 한 기능"**: P1~P12를 관통하는 *상위 원칙*. 본 분석은 이 원칙의 *결과*만 봤지 원칙 자체를 식별하지 못함. 다음 분석부터는 이걸 0번으로 두고 P1~P12를 *전술*로 본다.
- **P13 "라이팅은 디자인 시스템의 일부"**: 카피·보이스톤도 토큰처럼 관리됨. UI 톤만 보고 라이팅을 분리하던 본 분석의 가정은 부정확.
- **P14 "디자인 토큰의 자동화 파이프라인"**: Figma → GitHub → 멀티플랫폼 자동 생성. 토큰을 사람이 옮기는 비용을 제로화. 본 분석은 토큰을 추출했지만 **운영 메커니즘**까지는 못 봤음.
- **P15 "컴포넌트는 논증을 통과해야 산다"**: 즉 디자인 시스템 자체가 **자생적 가지치기 시스템**. 컴포넌트 다양성보다 정당성을 우선.

### 7.9 정합성 재검증 결론

- P4, P5, P10은 **외부 공식 자료로 강하게 검증됨** → 확신도 상향.
- P2, P11은 **외부 자료에서 매핑 못 찾음** → 본 분석의 가설로 명시 유지.
- **P0(한 화면 한 기능), P13~P15** 4개 정책을 새로 추가하면, 추출 모델이 12개 → 16개로 확장되며 토스 디자인의 *방법론적 구조*(원칙 → 시스템 → 자동화 → 거버넌스)까지 포괄.
