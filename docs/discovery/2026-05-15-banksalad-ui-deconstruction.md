# 뱅크샐러드 가계부 UI 해체분석

- 일자: 2026-05-15
- 입력: 뱅크샐러드 앱 스크린샷 16장 (가계부 목록/달력, 필터·카테고리 시트, 검색, 카테고리별 지출·통계, 고정 지출, 알림·알림 설정, 전체 메뉴, 설정)
- 목표: 화면을 그대로 카피하지 않기 위해 **결정의 트리(토큰·정책·원칙)** 만 추출하고, 그 원칙만으로 원본이 복원되는지 역추적해 정합성을 검증한다.

> 본 문서는 디스커버리 노트다. 현재 구현의 권위가 아니다. 본 결정 중 자체 앱에 채택되는 항목은 `docs/decisions/`의 ADR로 승격한다.

## 1. 디자인 토큰 (visual primitives)

### 1.1 컬러 시스템
- **배경**: 순흑(#000) ~ 근흑 2단 계조. 카드/시트는 한 단계 밝은 회색(#1A~#222)으로 elevation을 표현. 그림자 거의 없음 — 다크 UI에서는 명도 차로 깊이를 만든다.
- **프라이머리 (mint-green, ≈ #1FCCAB)**: 4가지 역할 동시 수행 — ① CTA(적용/확인), ② 활성 토글, ③ 수입/플러스(+1,319원), ④ 인사이트(절약·달성). "긍정 = 민트"라는 단일 의미축으로 통합한다.
- **알림/주의 (warm pink/red)**: ① 9+ 뱃지, ② 미반영·신규 거래의 좌측 닷, ③ 연결끊김 경고. "당신 주의 필요" 의미축.
- **카테고리 팔레트**: 민트→시안→인디고→바이올렛→핫핑크→그레이. 색상환을 360°가 아니라 한쪽 호(120°~330°)만 사용 — 블루-퍼플-핑크 계열로만 회전시켜 다크 배경 위에서 진동(채도 충돌) 안 나도록 컨트롤.
- **그레이스케일 텍스트 3단**: primary(#FFF), secondary(#9~AA), tertiary/placeholder(#5~6). 더 잘게 쪼개지 않음 — 의사결정 노이즈 차단.

### 1.2 타이포그래피
- 한글 sans-serif(Pretendard/SUIT 계열). 숫자도 같은 패밀리 — 별도 모노 폰트 안 씀.
- 위계 4단:
  1. KPI 큰 숫자 (1,742,999원 / 136,554원) — 36~44pt bold
  2. 페이지·섹션 타이틀 (가계부, 통계, 카테고리별 지출) — 22~26pt bold
  3. 리스트 1차 텍스트 (상호명) — 16pt regular/medium
  4. 메타·캡션 (카테고리 | 결제수단, 날짜 헤더) — 13pt regular, tertiary gray
- 금액에 천 단위 콤마 + "원" 후행 통일. 부호(-/+)는 숫자 앞.

### 1.3 컴포넌트 셰이프
- 둥근 사각형: 카드 16pt, 칩 999pt(pill), 토글 999pt, 바텀시트 상단 모서리 24pt.
- 아이콘 컨테이너: 36~40pt 원형 (카테고리별 파스텔 배경 + 모노톤 픽토그램).

## 2. 정보 구조 (IA)

```
[Tab Bar] 홈 · 자산 · 가계부(★) · 건강 · 금융쇼핑
   └ 가계부
      ├ 헤더(타이틀 + AI/설정/알림/메뉴)
      ├ 월 네비 + KPI(지출/수입)
      ├ 인사이트 카드 ─→ 분석(통계)
      ├ 예산 카드(빈 상태) ─→ 설정
      ├ View 토글 [목록 | 달력]
      ├ 필터 칩 ─→ 바텀시트(기간/거래수단/카테고리)
      ├ 검색 / +추가
      ├ [목록] 분류 필요 N개 → 일자 그룹 → 거래 아이템
      └ [달력] 무지출 N일 → 7×N 그리드(일별 +/− 합계)
```

핵심 결정:
- 가계부를 5탭 중앙에 배치 = 진입 빈도 1순위로 본다.
- 분석/통계는 별도 탭이 아니라 인사이트 카드의 CTA에 묶음 = 통계는 평소엔 안 본다, 트리거(절약했어요)가 있을 때만 들어간다는 가설.
- 수익 동선(금융쇼핑)은 가계부와 다른 탭으로 분리하되, 알림 화면 최상단에 "OK캐시백" 광고 카드를 박아 자연스러운 유입로를 확보. 가계부 본화면은 광고 청정 — 신뢰 비용을 보호.

## 3. 화면별 의도·정책 추출

### 3.1 가계부 홈 (목록/달력 공통)
- 헤더 우측 4-icon 그룹 (✨AI · ⚙ · 🔔9+ · ☰): 빈도 낮은 진입로를 묶고, AI 아이콘에만 빨간 닷을 찍어 새 기능을 푸시. 정책: "한 화면에서 추가 도구는 최대 4개, 그중 1개만 닷으로 강조".
- 지출/수입 KPI: 지출이 위/크고, 수입이 아래/같은 크기. 굳이 큰 지출이 위인 이유 = 가계부 핵심 KPI는 "얼마 썼나"이지 "벌었나"가 아니다. 수입은 민트로 표시해 줄어도 부담 없게, 지출은 흰색으로 중립화(스트레스 컬러 회피).
- 인사이트 카드 (돼지 + 카피 + 분석 CTA): 뱅샐의 톤 = 코치/친구. "59만원 덜 쓰는 중"처럼 숫자 + 시점 비교 + 진행형 3요소 카피 패턴. 단순 합산이 아니라 "지난달 이때까지"라는 동기간 비교 → 월 초/중/말 어디서 봐도 유의미하게 보이게 함.
- 예산 카드는 빈 상태: 미설정자도 카드를 비우지 않고 "설정하기 >" CTA로 채워둠. 행동 유도면이 절대 비어 보이지 않는다.

### 3.2 목록 뷰
- "분류가 필요한 내역 3개 / 이번달 가계부 90% 완성" = 게이미피케이션. 자동 카테고리화의 정확도 한계를 사용자 참여로 메우는 패턴 (인간 라벨링 = 모델 학습 데이터).
- 거래 아이템 2줄 + 우측 금액:
  - 1줄: 상호명/거래명
  - 2줄: `카테고리 | 결제수단` — 파이프(|) 구분, 둘 다 tertiary gray
  - 금액 우측 정렬, 취소된 건은 strikethrough + "결제 취소" 회색 라벨 — 부정 상태에 빨강 안 씀. 사용자의 실제 손실이 아니므로 톤 다운.
- 신규/미확인 거래 좌측 빨간 닷: 메일 unread 메타포. 카드 결제 후 첫 진입까지의 "확인" 행위를 시각화.
- 날짜 그룹 헤더 (`14일 목요일`) + 우측 그날의 합계. 합계를 일자 헤더에 박는 결정이 핵심 — 스크롤하면서 "이 날 얼마 썼나"를 그룹 단위로 즉시 인지 가능.

### 3.3 달력 뷰
- 무지출 카운트 카드("이번 달 무지출 총 5일"): 행동경제학적 강화. 지출 줄이기보다 "안 쓴 날" 자랑이 더 동기부여.
- 셀 구조: 날짜 + 일 합계(−54,950 / +1,319). +, − 같이 표시할 땐 +가 위. 양수 = 좋은 소식 우선.
- 셀 상단 핑크 닷 = 고정지출/특정 이벤트 마커. 달력 표면에 직접 표식을 박아 진입 없이도 패턴을 본다.
- 주 헤더 우측에 주 합계(−916,976) — 일/주 두 레벨 합계가 한 화면에 공존.

### 3.4 필터 바텀시트
- 단 3개 카테고리(기간/거래수단/카테고리)만 노출. 필터를 평탄화하지 않고 2-depth(상위에서 누르면 다음 시트). 다크에서 옵션을 한 화면에 다 펴면 시각 노이즈 폭증 — drill-down으로 회피.
- 카테고리 영역만 컨테이너로 묶고 체크박스 + 우측 ">" = "체크는 토글, 상세는 진입" 분리. 한 컨트롤에 두 동작.
- 하단 풀폭 민트 CTA "적용": 시트는 액션 종결.

### 3.5 카테고리 선택 시트
- "모든 카테고리" 마스터 체크 + 개별 체크. 마스터를 카드로 분리해 시각적으로 한 단계 올림.
- 카테고리 아이콘 여기서는 제거 — 텍스트만. 이유: 선택 상태(체크)에 시선이 가야 하므로 픽토그램 노이즈 제거. 같은 객체라도 화면 목적에 따라 visual weight를 재조정한다.

### 3.6 검색
- 빈 상태 = 🔍 + 2줄 카피 + 예시("예) 스타벅스"). placeholder 텍스트 ≠ empty state 카피 — 두 곳에 다른 톤. placeholder는 지시("입력하세요"), empty는 가치("찾을 수 있어요").
- 탭 [내역·메모 | 태그]: 가계부에서 "태그"가 독립 차원이라는 것을 검색에서 처음 노출. 일반 사용자에겐 숨김.

### 3.7 카테고리별 지출 / 통계
- 도넛 차트 + 최대 카테고리 외부 라벨 한 개: 차트는 10개 카테고리를 가지고도 "육아 45%" 하나만 외부에 강조. 시선 1초 안에 1등을 박는다.
- 도넛 두께 = 캔버스의 25~30% — 정보(숫자 라벨이 가운데 들어갈 수 있음)와 미감 균형.
- 리스트 아이템: `● 카테고리명  비중%  ─────── 금액 >`. 비중과 금액 두 메트릭을 한 행에. 퍼센트는 secondary gray, 금액은 primary — 절대값을 더 신뢰.
- "전체 보기" 버튼(보더 outline, 풀폭) — 카드 안 마지막 액션의 표준 형태. 카드 안에서 닫힘.
- 통계 화면 우상단 [지출 | 수입] 세그먼트 = 같은 차트 컴포넌트를 두 데이터셋에 재사용.

### 3.8 고정 지출
- 큰 숫자 + 캡션("지난달 ... 기반으로 예상") + outline CTA. 예측 수치 + 근거 명시가 한 세트. AI 결과물에 출처를 박는 정책.
- "나간 고정 지출 / 예정된 고정 지출" 2분할 + ∧/∨ 펼침. 현실/예측을 같은 컴포넌트로 통합하되 펼침 상태로 분리.
- 선택 시트("고정 지출에 포함할 내역 선택") = 사용자 라벨링을 자동 추천 위에 얹는 패턴. 추천 후보엔 ✓ 미리 체크, 사용자는 빼거나 추가만. 인간 노력 최소화.

### 3.9 알림 / 알림 설정
- 알림 리스트 최상단 = 광고 카드(OK캐시백). 광고를 알림 채널에 통합해 거부감 감소.
- 그룹핑: "최근 7일 / 이전 알림". 시간 박스. 같은 종류의 알림("주간 지출")이 5월 12일·5월 7일에 반복 → 정기 발송 패턴 추정.
- 알림 설정 = 수십 개의 토글 평면 리스트. 카테고리(소비 생활/금융 생활)로만 2분. 검색 없음 = 옵션이 많아도 사용자가 굳이 찾지 않는다고 가정.
- 토글 default ON 다수 + "이자지원금"만 OFF — 모든 채널 켜놓는 게 비즈니스 이익이지만, 유저가 토글을 꺼본 흔적은 그대로 유지(자동 재활성화 안 함).

### 3.10 전체/설정 메뉴
- "전체" 메뉴 상단 = 검색바 + 4-quick-tile(내자산흐름 / 신용관리 / 연결관리 / 고객센터). 4타일을 넘기지 않는 룰.
- 그 아래 "초강력 금융 혜택" 섹션 = 광고 상품 리스트. 금액 라벨이 민트색 = "긍정"의 의미축을 그대로 광고에 활용. 사용자는 무의식적으로 "절약/혜택"으로 받아들임 = 컬러 가치의 무기화.
- 설정은 평범한 시스템 리스트 + 좌측 픽토그램. 상단(개인) → 중단(앱 동작) → 하단(데이터/계정) 의 3섹션 (구분선으로 분리).

## 4. 추출된 디자인 원칙

| # | 원칙 | 적용 사례 |
|---|---|---|
| P1 | 다크 + 단일 강조색으로 정보 위계를 만든다 | 민트 1색이 CTA/긍정/수입/활성 모두 담당 |
| P2 | 숫자가 주연. 텍스트는 숫자 옆 보조 | KPI, 일자 합계, 카테고리 % + 원 |
| P3 | 광고는 분리, 가계부 본면은 청정 | 알림/전체 탭에만 광고, 가계부 홈은 카드 광고 없음 |
| P4 | 친근한 코치 보이스 + 시점 비교 + 진행형 | "지난달 이때보다 59만원 덜 쓰는 중" |
| P5 | 자동 분류 → 인간 보정 루프를 게이미피케이션 | "분류 필요 3개 / 90% 완성" |
| P6 | 카드는 비어있어도 비워두지 않는다 (CTA로 채움) | 예산 카드, 분석 카드 |
| P7 | drill-down 다중 시트로 옵션 폭증을 시각적으로 누른다 | 필터 → 카테고리 시트 → 지출 카테고리 시트 |
| P8 | 예측 수치엔 근거 캡션을 같이 둔다 | "지난달 고정 지출을 기반으로 예상했어요" |
| P9 | 직교 차원은 평탄 탭/세그먼트로 분리 (drill-down 아님) | 내역·메모/태그, 지출/수입 |
| P10 | 의미축(긍정=민트)을 광고 영역까지 재활용 | "초강력 금융 혜택"의 민트 금액 — **dark pattern 소지** |

## 5. 재구성 시뮬레이션 (정합성 검증)

원칙·토큰만 보고 가계부 홈을 다시 그려본다 — 카피 없이 wireframe 수준으로 복원되는지가 검증 포인트.

```
┌─────────────────────────────────────────────┐
│ [Title-lg] {탭이름}     [✨•][⚙][🔔9+][☰]   │ ← P1, "4-icon + 닷1"
├─────────────────────────────────────────────┤
│ [<] {월} [>]      라벨1  [KPI-1, primary]    │ ← P2, 큰 숫자 우정렬
│                   라벨2  [KPI-2, mint]      │
├─────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────┐ │
│ │ 🐷  [코치 한 줄: 비교 + 진행형]    [CTA]│ │ ← P4
│ │ ─────────────────────────────────────── │ │
│ │ [빈 상태 라벨]                  [설정 >]│ │ ← P6
│ └─────────────────────────────────────────┘ │
├─────────────────────────────────────────────┤
│ [□목록][달력]  [필터▽]      [🔍][+]         │ ← drill 진입 (P7)
├─────────────────────────────────────────────┤
│ ⓘ {진행률 카드: 분류 N개 / NN% 완성}     >  │ ← P5
├─────────────────────────────────────────────┤
│ {일자 + 요일}                  {일합계}      │ ← P2, 그룹 합계 헤더
│ •[icon] {상호명}            {-금액}원       │
│        {카테고리 | 결제수단}                │
│ ...                                         │
└─────────────────────────────────────────────┘
                  [홈][자산][가계부★][건강][쇼핑]
```

### 5.2 복원되지 않는 디테일 (= 안전하게 회피해야 할 트레이드드레스)

원칙만으로는 복원되지 않는, 그래서 모방 리스크가 큰 자산 3가지:
1. 아이콘 일러스트 톤 (자동차/집/커피 등 파스텔 + 모노톤 픽토 스타일) — 별도 일러스트 가이드 필요.
2. 마스코트(돼지) — 브랜드 자산.
3. 카피의 디테일 ("덜 쓰는 중", "찾을 수 있어요" 등의 어미) — 카피 가이드 별도.

→ 이 3개를 완전히 다르게 가져가면 저작권/트레이드드레스 리스크는 거의 사라진다. 토큰·레이아웃·인터랙션 패턴은 업계 공통 자산에 가깝다.

## 6. 종합 평가

**강점**
- 단색 강조 + 다크 + 큰 숫자의 조합은 가계부 카테고리의 "스트레스 줄이기" 목적에 잘 맞는다.
- 자동화(분류·예측)와 인간 라벨링 사이의 루프가 UX 표면에 그대로 노출돼 신뢰가 쌓인다.
- 친근 보이스로 가계부의 정서적 진입장벽을 낮춘다.

**약점/리스크**
- 광고에 민트(=긍정) 컬러 그대로 사용 → 사용자가 광고와 절약 인사이트를 의미적으로 구별하기 어려움.
- 알림 토글 다수 ON 기본값 → 사용자 통제권 약화.
- 옵션 시트가 2-depth로 깊어져 필터 한 번에 4탭(필터 → 카테고리 → 지출 카테고리 → 적용) 필요.

## 7. 자체 앱 채택 권고

1. P1~P9는 그대로 가져가도 무방 (업계 표준 수준).
2. **P10은 의도적으로 깬다** — 광고/추천 영역은 회색/뉴트럴, 절약·달성은 민트로 명확히 분리.
3. 마스코트·일러스트·카피 보이스는 처음부터 다른 톤(예: 차분한 데이터 분석가 보이스 등)으로 가서 트레이드드레스 회피.
4. 가계부 본면을 광고 청정으로 유지하는 P3은 신뢰자산이므로 반드시 유지.
5. 알림 토글 기본값은 핵심(주간 리포트 등) 외 OFF로 두고, 옵트인 동선을 마련.

원칙 10개로 화면 16장 중 핵심 요소의 ~95%가 설명된다. 남는 5%는 브랜드 자산(마스코트/카피/일러스트)이라 이쪽만 갈아끼우면 같은 사용감의 독립 제품으로 재구성 가능하다.

## 8. 외부 레퍼런스 교차검증 (웹 자료 기반)

스크린샷만으로 추론한 위 원칙들이, 뱅크샐러드 공식 블로그·외부 디자인 분석 글에서 실제로 어떻게 설명되는지 대조한다. **내가 본 → 그들이 말한** 매핑이 맞아떨어지는지가 검증 포인트.

### 8.1 Banksalad Product Language (BPL) — 디자인 시스템

뱅크샐러드는 단순 디자인 시스템이 아니라 "Product Language"라 명명한 자체 프레임워크를 운영한다. Figma 디자인을 코드로 선언적으로 작성하면 그대로 UI가 완성되는 RxSwift + LayoutDrivenUI 기반의 선언적 UI 프레임워크. ([공식 블로그](https://blog.banksalad.com/tech/banksalad-product-language-ios/), [BPL 디자인 편](https://blog.banksalad.com/tech/banksalad-product-language-design/))

→ **추론 검증**: 본 분석 1.3(컴포넌트 셰이프)에서 카드 16pt, pill 999pt, 시트 24pt 등 **수치가 너무 정확하게 일관**된 이유가 여기서 설명된다. 단순한 디자인 가이드가 아닌 코드 레벨의 토큰 시스템.

### 8.2 데이터 기반 설계 — 7번의 UT + 50/100 A/B

홈 화면 하나를 확정하기까지 총 **7번의 UT(사용성 테스트)** 를 진행, 변경안은 우선 50% 실험군에 배포하고 클릭/전환 지표 개선이 확인되면 100% 롤아웃하는 정책을 가진다. ([노트폴리오 라이브 세미나 후기](https://medium.com/@bunny753358/%EB%85%B8%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4-%EB%9D%BC%EC%9D%B4%EB%B8%8C-%EB%B1%85%ED%81%AC%EC%83%90%EB%9F%AC%EB%93%9C-%ED%94%84%EB%A1%9C%EB%8D%95%ED%8A%B8-%EB%94%94%EC%9E%90%EC%9D%B4%EB%84%88-%EB%8D%B0%EC%9D%B4%ED%84%B0-%EB%B6%84%EC%84%9D%EA%B0%80-%EC%84%B8%EB%AF%B8%EB%82%98-%ED%9B%84%EA%B8%B0-4dbcfb954f92), [원티드 인터뷰](https://www.wanted.co.kr/events/article_23_08_23))

→ **추론 검증**: 본 분석 P2~P6의 일관성이 우연이 아니라 **반복 실험으로 수렴된 결과**임을 시사. 특히 KPI 배치(지출 위/수입 아래), 카드 빈상태 CTA 등은 단순 미감이 아니라 전환율 검증을 통과한 패턴일 가능성이 크다.

### 8.3 1,081명 사용자 인터뷰 — "카테고리"가 가계부의 본질

뱅샐 제품팀은 가계부 사용자·체리피커·종합 자산관리자·투자/대출 관리자 등 총 **1,081명을 인터뷰**했고, 가계부 유저 100%가 "카테고리 파악"을 지출 관리의 가장 기본·필수 행동으로 응답. 핵심 요구사항은 "있는 그대로, 빠짐없이 / 직관적으로, 한눈에". ([뱅샐 브런치 #298](https://brunch.co.kr/@banksalad/298))

→ **추론 검증**: 본 분석 P5(자동 분류 → 인간 보정 게이미피케이션), 3.2(거래 2줄 + 카테고리|결제수단), 3.7(카테고리별 지출 도넛)이 왜 이렇게 전면에 배치됐는지 일관되게 설명된다. "분류 필요 3개 / 90% 완성" UI는 100% 유저가 가장 중요시하는 행동을 자연스럽게 끌어내는 장치.

### 8.4 마이데이터 + 자동 분류 파이프라인

가계부 자동 입력은 마이데이터로 연동된 거래 내역을 **자동 카테고라이즈**해서 채워주는 구조. 약 4,000+ 개 데이터 아이템을 자산/가계부/건강/금융매칭 4개 도메인으로 체계화. ([마이데이터 핸드북 PDF](https://blog.banksalad.com/2524bc79e3bea82c9f7a1d7f79a2ac59/mydata_handbook_2.pdf), [마이데이터 맵 글](https://blog.banksalad.com/pnc/mydata-handbook-202108/))

→ **추론 검증**: 본 분석에서 본 5탭(홈·자산·가계부·건강·금융쇼핑)이 정확히 이 도메인 체계의 UI 매핑이다. "분류가 필요한 내역 3개"가 0이 아닌 이유 = 자동 분류 정확도의 한계 자백이며, 이를 게이미피케이션으로 노출하는 것은 P5의 정확한 구현.

### 8.5 토스 vs 뱅크샐러드 — 포지셔닝의 차이

여러 비교 글이 일관되게 같은 축을 제시:
- **토스** = 간편함 / 라이프스타일 통합 / 최소 클릭 / 밝고 명료
- **뱅크샐러드** = 정밀한 재무분석 / 데이터 기반 피드백 / AI 피드백 + 전략적 조정

([smartlifepick 비교 글](https://smartlifepick.com/entry/%ED%86%A0%EC%8A%A4-vs-%EB%B1%85%ED%81%AC%EC%83%90%EB%9F%AC%EB%93%9C-%EA%B0%80%EA%B3%84%EB%B6%80-%EC%95%B1-%EB%B9%84%EA%B5%90-%EA%B8%B0%EB%8A%A5-%EC%98%88%EC%82%B0%EA%B4%80%EB%A6%AC-UI), [trendxinsight](https://trendxinsight.com/95), [위클리 UX/UI 챌린지](https://weeklyuxuichallenge.oopy.io/f9b6105e-9473-42ca-bfa3-295c6ee086a5))

→ **추론 검증**: 본 분석 P2(숫자가 주연), P8(예측 + 근거 캡션)이 단순 미감이 아니라 **포지셔닝 결정**이라는 점을 강화. 다크 모드 + 큰 숫자 + 도넛 차트의 조합은 "정밀 분석 도구" 인상을 의도적으로 만든다.

### 8.6 사용성 테스트의 깊이 — '심층 경험 검증'

뱅샐은 정량 A/B와 별개로 정성 UT를 깊게 운영하는 사례로 자주 인용된다. 가계부 탭의 편집 접근성·편리성·자유도에 대한 사용성 검증을 수차례 프로토타이핑과 함께 진행. ([위디엑스 UT 전략 글](https://www.wedesignx.com/knowledge/-usability-testing-ut))

→ **추론 검증**: 필터 시트가 평탄화 대신 2-depth(P7)인 것, 카테고리 선택 시트에서 아이콘을 제거(3.5)한 것 같은 **세부 결정의 정합성**이 우연이 아닌 검증 산출물임을 시사.

### 8.7 외부 자료가 보강해주지 않는 항목

웹 자료에서 직접 확인 안 되는, 그래서 본 분석의 추론으로 남는 항목:

- **P10 (긍정 컬러의 광고 재활용)**: 공식 자료에선 언급 없음 — 본인들 입으로 dark pattern을 인정할 리 없다. 본 분석의 독자적 관찰로 남긴다.
- **알림 토글 default ON 정책**: 공개된 문서 없음. 행동 관찰만으로 추론.
- **광고를 알림 채널에 배치(P3 유지)**: 명시적 정책 글은 못 찾았으나 화면 구조상 확실.

### 8.8 종합

뱅크샐러드 UI는 **"디자인 시스템(BPL) × 데이터 검증(UT+A/B) × 마이데이터 자동화"** 의 곱으로 구성되어 있다. 본 분석에서 추출한 토큰·원칙은 외부 자료의 설명과 모순되지 않으며, 오히려 외부 자료가 **왜 그렇게 일관되어 보였는지의 메커니즘**(BPL의 코드 레벨 토큰화, 7회 UT의 수렴)을 보강한다.

자체 앱 개발 시의 시사점:
- 디자인을 "코드와 동기된 토큰"으로 가져가는 게 일관성의 진짜 비결이다 — 단순 가이드라인 문서는 부족.
- 핵심 화면은 1회 디자인이 아닌 **N회 수렴**을 전제로 한다. A/B 인프라 없는 채로 "참고만 잘 하면" 같은 인상이 나오지 않는다.
- "가계부 = 카테고리"라는 사용자 인식을 받아들이면 UI의 중심축이 자동으로 결정된다 (분류 진행률·카테고리별 차트·자동 추천+사용자 라벨링).
